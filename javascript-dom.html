Javascript and the DOM



JavaScript and its relationship to HTML and CSS


My favorite analogy of the relationship between JavaScript, HTML and CSS is that if a web page were a person, HTML would be the words the person says, CSS would be the clothes and accessories the person wears, and JavaScript would be the person's brain, the words they say are thought through by the brain and imply the logic of the person's mind.



Control flow and loops


The control flow is the order in which the computer executes statements in a script. When a computer gets a script, it ‘read’ the script from top to bottom. As for the loops, they are used in JavaScript to perform repeated tasks based on a condition.

If I were to think of an analogy, control flow is like a cooking process where we first need to prepare the ingredients for the dish, then sautéed, then served on a plate, and then set on the table. And loops are like the process of, for example, stir-frying, keep stir-frying until the condition is met. What condition? That is when everything is cooked! Then, we can proceed to the next step, serving it on the plate.



The DOM and how to use it


DOM is short for Document Object Model. It is the way a web browser represents a web page internally. For me, it’s like the web browser using different tags to translate the html script to a web page. We call it the DOM tree because it kind of looks like a tree upside down.


We can use a thing called DevTools to interact, or we could say manipulate, the web pages. In order to do that we can right click on a web page and click inspect, or press the F12 key.
